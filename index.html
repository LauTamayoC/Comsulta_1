¿Cómo funciona Internet?

HTTP (Protocolo de transferencia de hipertexto)

Establece un código para que el servidor solicitante y el que contiene la información solicitada puedan hablar un mismo idioma a la hora de transmitir información por la red.

Este protocolo establece métodos para enviar solicitudes (denominados "verbos") y tiene cierta flexibilidad para adaptarse a nuevas solicitudes y funciones, particularmente a medida que evoluciona en sus versiones.

Se basa en petición-respuesta entre el servidor y el usuario que realiza la solicitud de transmisión de datos.

TCP/IP (Protocolo de control de transmisión/protocolo de Internet)

Clasifica los protocolos de red, describiendo el conjunto de guías y reglas generales que hacen posible que un equipo sea capaz de intercambiar información en una red.

FTP (Protocolo de Transferencia de Archivos)

este protocolo nació antes de que existieran Internet o el correo electrónico, ya que fue uno de los componentes básicos de ARPANET, Este protocolo funciona entre ordenadores que estén conectados a una red TCP, que significa Transmission Control Protocol o Protocolo de control de transmisión, Las conexiones FTP tienen una relación de cliente y servidor. Esto quiere decir que un ordenador tiene que estar configurado como servidor FTP, ese en el que se aloja el contenido, y luego tú te conectas a él como un cliente, el cliente FTP más popular entre la mayoría de los usuarios sea FileZilla de Mozilla, los mismos creadores que Firefox

DNS (Sistema de nombres de dominio)

El protocolo DNS (Sistema de Nombres de Dominio) desempeña el papel de un directorio de traducción en internet. En lugar de que las personas recuerden números difíciles de entender llamados direcciones IP, el DNS permite que las personas utilicen nombres de dominio amigables, como "www.ejemplo.com", para acceder a sitios web. Funciona como una especie de intermediario: cuando alguien introduce un nombre de dominio en su dispositivo, el protocolo DNS trabaja en segundo plano para encontrar la dirección IP correspondiente de ese dominio. De esta manera, las computadoras pueden comunicarse entre sí utilizando números, mientras que los usuarios pueden navegar la web con nombres fáciles de recordar

WWW (World Wide Web)

El navegador entonces impulsa una serie de comunicaciones para obtener los datos de la página solicitada. El primer paso consiste en transformar el nombre del servidor de la URL en una dirección IP, utilizando la base de datos conocida como DNS. Luego se envía una solicitud HTTP al servidor a fin de lograr acceder al recurso. Primero se pide el texto HTML y después se produce el análisis por parte del navegador

CDN

Es una red de servidores conectada que permite cargar las páginas web para las aplicaciones que utilizan muchos datos. CDN puede ser una red que entrega contenido o distribuye contenido. Al visitar un sitio web, los datos del servidor del sitio web deben viajar a través de Internet para llegar a la computadora del usuario. Si el usuario se encuentra lejos de ese servidor, un archivo de gran tamaño, como un video o una imagen del sitio web, se demorará mucho en cargar. En su lugar, el contenido del sitio web se almacena en servidores de CDN ubicados geográficamente más cerca de los usuarios, por lo que el contenido llega a sus computadoras mucho más rápido.

HTML

(Lenguaje de Marcado de Hipertexto) en inglés (HyperText Markup Language), Es el lenguaje utilizado para crear y estructurar contenido en la web.HTML permite a los desarrolladores web definir la estructura y el contenido de una página web, como el texto, las imágenes, los enlaces y otros elementos, de manera que los navegadores puedan interpretar y mostrar el contenido de manera coherente y comprensible para los usuarios. HTML utiliza "etiquetas" o "marcas" para definir diferentes elementos en una página web. Cada etiqueta tiene un propósito específico, como indicar encabezados, párrafos, imágenes, enlaces y más. Las etiquetas son interpretadas por los navegadores web para mostrar el contenido de la página de manera visual y organizada. Además de definir la estructura y el contenido de una página, HTML también puede trabajar en conjunto con otros lenguajes y tecnologías, como CSS (Hojas de Estilo en Cascada) para el diseño visual y JavaScript para agregar interactividad y funcionalidad a las páginas w
